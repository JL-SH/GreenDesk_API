1. Implementar Alembic 
    Motivo:
        ¿Por qué create_all es para principiantes?
        Base.metadata.create_all(bind=engine) solo hace una cosa: si la tabla no existe, la crea.

        El problema: Si mañana decides añadir una columna price a tu tabla devices, create_all no hará nada. No sabe modificar tablas existentes.

        La solución (Alembic): Es como Git para tu base de datos. Registra cada cambio (añadir columnas, cambiar tipos, borrar índices) en archivos de "migración" que puedes subir al repositorio. Así, todo tu equipo tiene la misma estructura de base de datos.
